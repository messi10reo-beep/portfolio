# 凪 NAGI — 整体・ボディケアサロン向け AI業務改善デモ

架空のボディケアサロン「**凪 NAGI**」を題材に、**予約・FAQ・問い合わせ対応を一体化**した小規模事業者向けのAI業務改善パッケージのデモです。
単なるLPではなく、「**集める → AIで整理する → 人が確認して返す**」という現実的な運用フローを、**APIキーなし・完全フロント完結**で形にしています。

> ⚠️ これは架空の店舗を題材にした**ポートフォリオデモ**です。実在の店舗・医療行為とは関係ありません。AIの回答は一般的なFAQ補助に限定し、医療的な断定は行いません。最終確認は必ず人（スタッフ）が行う設計です。

---

## 作成目的

- 文系学生が、**AIをビジネス課題の解決に使える**ことを示す
- 小規模事業者の課題をヒアリングし、**Web・予約導線・AI活用までを一気通貫**で提案できることを示す
- 「アプリを作れる」ではなく、**課題 → 提案 → 制作物 → 成果**の形で説明できることを示す
- AIで何でも自動化するのではなく、**人間が確認すべき部分も設計**できることを示す
- 就活（長期インターン応募）／ランサーズ・受託営業の両方で使えるポートフォリオにする

## 使用技術

- **HTML / CSS（和モダン・モバイル最優先）**
- **Vanilla JavaScript**（DOM構築は `textContent` ベースでXSS安全、静的SVGのみ `insertAdjacentHTML`）
- **localStorage** による簡易データ保存（外部送信なし）
- **簡易RAG風FAQ検索**（キーワード重みづけ＋根拠提示）
- **ルールベース分類**（説明可能なAI風ロジック）
- 外部APIキー不要 / 著作権素材に依存しない（配色・グラデーション・アイコンで表現）

### データ設計（型）

```
Booking : id, name, email, phone, menu, firstChoice, secondChoice,
          symptoms, isFirstVisit, note, category, aiSummary, createdAt, status
Inquiry : id, name, email, type, message, category, aiSuggestedReply, createdAt, status
FAQItem : id, category, question, answer, keywords
```

## セットアップ方法

ビルド不要。静的ファイルのみで動作します。

```bash
# リポジトリ直下（portfolio/）で簡易サーバを起動
python3 -m http.server 8000
# ブラウザで開く
open http://localhost:8000/massage/
```

> `file://` で直接開いてもおおむね動作しますが、相対パスの都合上 **HTTPサーバ経由での閲覧を推奨**します。本番は Vercel（静的ホスティング）にそのままデプロイできます。

## デモの見どころ

1. **予約フォーム**：症状に「肩こり」「腰痛」「猫背」などを書いて送信すると、AIがカテゴリを判定し、当日の確認ポイントを提示します。
2. **AI FAQ**：「初回はいくら？」「服装は？」などを質問すると、根拠（FAQ項目）と信頼度つきで回答。答えにくい質問は問い合わせへ誘導します。
3. **問い合わせ**：種別と本文から自動分類し、**返信文のたたき台**を生成（コピー可能）。
4. **管理ダッシュボード**：上記の送信内容が一箇所に集約され、AI分類・未対応件数・今日の対応リストが見えます。**デモデータのリセット**も可能。
5. すべて **APIキーなし**で動作し、入力は **ブラウザ内のみ**に保存されます。

## ポートフォリオとしてのアピールポイント

- AIを「完全自動返信」ではなく、**人間の確認を前提にした補助（半自動）**として設計
- **医療的な断定を避け**、安全なFAQ支援に範囲を限定
- 小規模事業者でも運用しやすい**シンプルな画面設計**
- **課題・提案・制作物・成果**を明確化し、採用担当・受託の見込み客の双方が短時間で価値を理解できる
- 説明可能なルールベース＋簡易RAGで、**「なぜその回答／分類になったか」を説明できる**
