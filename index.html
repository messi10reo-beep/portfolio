機密文書QA — ローカル完結RAG（評価駆動）

製造業など「社外に出せない文書」を想定した完全ローカルRAG。埋め込み・検索・生成をすべて端末内で実行し、外部APIに一切データを送らない設計（オフライン動作・APIコスト0）。評価基盤（npm run eval）で検索・生成品質を数値化し、型番の取り違えを完全一致ブーストで解消。BM25全文融合やリランカは「評価して効かないので既定OFF」——流行りの手法を盛るのではなく、測って選ぶ開発を実践しています。LLMは抽象化し、商用API（Claude/OpenAI）へ1行で差し替え可能です。

設計の深掘り — 何をどう判断したか▾
  • ハイブリッド検索：ベクトル検索だけでは型番など固有記号を取り違えた。完全一致ブーストを併用して検索Recall@1を85→100%に改善。
  • 評価を先に設計：質問セットで検索・生成品質を毎回自動計測（npm run eval）。改善は体感でなく数値で判断する。
  • 測って捨てる：BM25全文融合・リランカは実装したうえで評価し、効果がなかったため既定OFF。流行を理由に採用しない。
  • 差し替え可能なLLM層：ローカルLLM（Ollama）前提だが、抽象化により商用API（Claude/OpenAI）へ1行で切替可能。要件に応じて選べる。

凪 NAGI — 整体サロンのAI業務改善スイート

「小規模店舗が予約・問い合わせ対応に追われ本業に集中できない」という課題を自分で見つけ、企画から実装・公開まで一人でやり切ったAI業務改善スイート。架空のボディケアサロン「凪 NAGI」を題材に、AI事前整理つき予約フォーム／根拠つきAI FAQチャット／自動分類＋返信たたき台つき問い合わせ／それらを集約する管理ダッシュボードを実装。AIは「人の確認を前提にした半自動」設計です。課題→提案→制作物→成果をケーススタディで読めます。

設計の深掘り — 何をどう判断したか▾
  • 半自動の設計：AIの役割は「分類と下書き」まで。予約の確定や返信の送信は必ず人が行うワークフローにし、現場が安心して使える形にした。
  • セキュリティの実践：公開後の自己監査でDBの行レベルセキュリティ（RLS）が全開放なことを発見。最小権限に締め直し、管理画面をパスコードからSupabase Auth（メール+パスワード）に置き換えた。
  • デモとしての割り切り：体験デモ部分はlocalStorage完結・外部送信なし・APIキー不要。誰でもすぐ触れることを優先した。
  • 型化と横展開：「サイト＋予約＋管理」の共通の型に整理し、飲食・美容室・ネイルの3業種へ展開。同じ設計が業種を跨いで通用することを確認した。

G検 Trainer — G検定 学習アプリ

挑戦中のJDLA G検定（ディープラーニング検定）に合格するため、学習そのものを仕組み化して自作したアプリ。シラバス2024準拠の8章で要点ノート・暗記カードからインプットし、全320問の演習（章別／模試／弱点復習）でアウトプット。全問に根拠つき解説を実装し、進捗・正答率を自動記録。React + TypeScriptで単一HTMLにビルドしたPWAで、学習データは端末内のみ（localStorage完結）です。

02 — Approach

開発の進め方

流行りの手法を盛るのではなく、課題から出発して測って選ぶ。すべてのプロジェクトをこの進め方で作っています。

1
課題を見つける経営学部で学ぶ業務の視点から「人手に頼っていて、仕組みで軽くできる仕事」を探します。凪NAGIは店舗の予約・問い合わせ対応の負担から始まりました。
2
小さく設計する最小構成で価値を検証できる形に切り出します。機密データを扱う想定では、外部に出さないローカル完結を基本にします。
3
実装するTypeScript strict・テスト前提。AIは全自動にせず「人の確認を前提にした半自動」に設計し、現場が安心して使える形にします。
4
測って判断する評価基盤で検索・生成品質を数値化し、改善は数値で確認します。BM25融合やリランカのように「効かない手法は流行でも既定OFF」にします。
5
公開して運用する全プロジェクトを誰でも触れる状態で公開し、テスト・CIで品質を保ちながら改善を続けます。
  • ✓
    動くもので証明する口頭やスライドでなく、実際に操作できるプロジェクトで判断してもらいます。
  • ✓
    盛らない・正直にできること／できないことを率直に。数値や成果を誇張しません。
  • ✓
    小さく始めて確かめるまず小さく作って効果を確認し、必要な分だけ拡張します。
  • ✓
    AIは人の確認を前提に全自動でなく半自動。最終判断は人が行う、安全な設計にします。
  • ✓
    データに配慮した実装外部に出さないローカル完結の構成・入力チェック・型安全な実装。
